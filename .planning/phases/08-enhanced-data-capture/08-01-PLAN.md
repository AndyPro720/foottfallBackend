# Plan 08-01: Status Tracking & Property Info Enhancements

## Objective
Implement a robust property status workflow (Occupied, Available, Under Construction) with conditional sub-fields for OC states and timeline. Move Carpet Area.

## Tasks
1. Update `propertyFields.js`: add `propertyStatus` select (Occupied, Available, Under Construction) to `propertyInfo`. Add conditional numeric fields `completionTime` (months), `partOC`, and `completeOC` beneath it.
2. In `propertyFields.js`, add `size` (Carpet Area) to the `propertyInfo` section and remove it from `specs`.
3. Update `IntakeForm.js`: modify `renderSelect` and the event listeners to watch `propertyStatus`. If value is "Under Construction", unhide `completionTime`, `partOC`, and `completeOC` inputs.
4. Update `PropertyDetail.js`: display the custom status logic correctly instead of the hardcoded Active/Inactive loop.

## Verification
- Status dropdown displays correct options. Selecting 'Under Construction' reveals OC and time fields.
- Form submits and saves these new values correctly. 
- Carpet area shows up in the first section.
