<objective>
Add project creation flow to the IntakeForm page (toggle between Property/Project mode), display projects on the Home page with distinct card styling, and create a Project Detail view showing common info + unit list with "Add Unit" capability.
</objective>

<must_haves>
- IntakeForm has a mode toggle (Property vs Project) at the top
- Project mode shows only PROJECT_SECTIONS fields
- Home page displays projects with distinct visual treatment (project badge/icon)
- Clicking a project navigates to `#project/{projectId}` route
- Project detail view shows project info at top and unit cards below
- "Add Unit" button in project detail view navigates to `#add?projectId={id}`
- Adding a unit from project context auto-populates project fields as read-only
- Hash routing supports `#project/{id}` and `#add?projectId={id}`
</must_haves>

<tasks>

<read_first>
- src/pages/IntakeForm.js
- src/config/propertyFields.js

<action>
Modify `renderIntakeForm` in IntakeForm.js:

1. At the top of the function, parse URL for projectId: `const urlParams = new URLSearchParams(window.location.hash.split('?')[1] || ''); const projectId = urlParams.get('projectId');`

2. Add a mode toggle at the top of the form (before sections), only if NOT adding a unit under a project:
```html

  <button type="button" class="mode-btn active" data-mode="property">Property</button>
  <button type="button" class="mode-btn" data-mode="project">Project</button>
```

3. Import `PROJECT_SECTIONS, UNIT_ONLY_SECTIONS, PROJECT_FIELD_NAMES` from propertyFields.js and `createProject` from projectService.js and `getProjectById` from projectService.js.

   - Render `PROJECT_SECTIONS` instead of `SECTIONS`
   - Change submit button text to "Create New Project"
   - On submit, call `createProject(data)` instead of `createInventoryItem(data)`
   - Navigate to `#` on success

5. When `projectId` is present in URL (adding unit under project):
   - Hide the mode toggle
   - Fetch project data via `getProjectById(projectId)`
   - Show a read-only "Project" banner at top: `Adding unit to: {projectName}`
   - Render `UNIT_ONLY_SECTIONS` for the form fields
   - On submit, add `projectId` to the data, also copy project-level fields (name → stored as projectName on the unit, plus city, tradeArea, buildingType, location, googleMapsLink for search/filter compatibility)
   - Call `createInventoryItem({...unitData, projectId, name: unitName || projectName})`

6. Wire up mode toggle button click handlers to re-render the form sections.

7. For the creatable-select fields (city, tradeArea) in project mode, use the same `initCreatableSelect` pattern as existing code.
</action>
