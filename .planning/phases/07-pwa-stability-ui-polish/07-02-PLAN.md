# Plan 07-02: Offline Auth Persistence Fix

## Objective
Ensure that when the PWA is reloaded or reopened offline, the user is not signed out. This requires shifting the `syncUserProfile` function to use a cache-first approach so it doesn't fail or return null when the server is unreachable.

## Tasks

<task id="1">
<title>Make syncUserProfile cache-first</title>
<read_first>
- src/backend/userRoleService.js (lines 8-31: syncUserProfile, and lines 36-63: getCurrentUserProfile which already uses cache-first)
</read_first>
<action>
In `src/backend/userRoleService.js`:

1. Update `syncUserProfile(user)` to prioritize cached data, just like `getCurrentUserProfile` does.
2. The logic should be:
   - Try `getDocFromCache(userDocRef)`.
   - If cache hits, execute a non-blocking background `setDoc` for `lastLogin: serverTimestamp()` (wrap in `try/catch` and don't await so it doesn't block offline).
   - Return the cached data immediately.
   - If cache misses, try `getDoc(userDocRef)`.
   - If `getDoc` fails (network error) AND cache missed, return a safe fallback object `{ uid: user.uid, role: "agent", status: "pending" }` instead of failing, to keep the user signed in offline.
   - If `getDoc` succeeds but document doesn't exist, create the new user record as it does now.
</action>
<acceptance_criteria>
- `syncUserProfile` contains `getDocFromCache(userDocRef)`
- `syncUserProfile` has a fallback return `return { uid: user.uid, role: 'agent', status: 'pending' };` to prevent offline lockouts
- Background `lastLogin` update does not block the return value
</acceptance_criteria>
</task>

## Verification
- Stop Vite dev server or use DevTools Network tab to simulate offline.
- Reload the app while authenticated and offline.
- The app should render cached content and NOT redirect to the login screen.

## must_haves
- `syncUserProfile` does not throw unhandled errors when offline
- Cache hits skip the server `getDoc` call
