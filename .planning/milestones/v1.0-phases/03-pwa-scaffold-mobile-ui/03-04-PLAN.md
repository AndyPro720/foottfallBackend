# Plan 03-04: Design System Overhaul — Linear/Stripe-Inspired

> **Wave 2** — Visual quality upgrade. Depends on Wave 1 (nav/manifest must work first).

| Gap | Source | Severity |

## Design Direction

Upgrade from basic glassmorphism to a **modern SaaS dashboard** inspired by Linear and Stripe:

- **Color palette evolution:** Keep the green/white brand identity but add depth with layered surfaces (dark → darker → darkest greens) instead of a single flat `#0a3622`. Add a subtle green gradient background.
- **Card styling:** Multiple surface elevation levels. Cards get subtle borders with `rgba(255,255,255,0.08)`, soft inner glow, and proper shadow layering.
- **Typography:** Tighter line heights, sharper weight contrasts (400 for body, 500 for labels, 600 for headings, 700 for display). Use `letter-spacing: -0.02em` on headings for that Linear feel.
- **Spacing rhythm:** Strict 4px grid. Utility classes for all spacing. No inline styles.
- **Animations:** `@keyframes` for card entrance (fade-up), skeleton shimmer, route transitions, button press feedback.
- **Form inputs:** Floating labels or crisp bordered inputs with focus rings (green glow).
- **Status indicators:** Colored dots/badges for inventory status instead of plain text.

**File:** `src/style.css` — Full rewrite.

**New token structure:**
```css
:root {
  /* Surfaces — layered depth */
  --bg-base: #061c14;
  --bg-surface: #0a3622;
  --bg-raised: #0d4a2f;
  --bg-overlay: rgba(255, 255, 255, 0.06);
  --bg-input: rgba(0, 0, 0, 0.25);
  
  /* Brand */
  --accent: #ffffff;
  --accent-hover: rgba(255, 255, 255, 0.9);
  --accent-green: #34d399;
  --destructive: #ef4444;
  
  /* Text */
  --text-primary: #ffffff;
  --text-secondary: rgba(255, 255, 255, 0.6);
  --text-tertiary: rgba(255, 255, 255, 0.4);
  
  /* Borders */
  --border-default: rgba(255, 255, 255, 0.08);
  --border-hover: rgba(255, 255, 255, 0.15);
  --border-focus: rgba(52, 211, 153, 0.5);
  
  /* Spacing — strict 4px grid */
  --space-xs: 4px;
  --space-sm: 8px;
  --space-md: 16px;
  --space-lg: 24px;
  --space-xl: 32px;
  --space-2xl: 48px;
  --space-3xl: 64px;
  
  /* Typography */
  --text-xs: 12px;
  --text-sm: 14px;
  --text-base: 16px;
  --text-lg: 20px;
  --text-xl: 24px;
  --text-2xl: 32px;
  
  /* Radii */
  --radius-sm: 6px;
  --radius-md: 10px;

```

**Required CSS classes (no inline styles allowed after this):**

Layout:
- `.page-header` — top section with title + subtitle
- `.page-content` — main scrollable area

Cards:
- `.card` — base card (replaces `.glass-card`). Uses `--bg-raised` with border, subtle box-shadow, and hover lift.
- `.card-header`, `.card-body`, `.card-footer` — card sections
- `.card-interactive` — adds hover state (border brightens, subtle translateY)

Forms:
- `.form-group` — label + input wrapper with spacing
- `.form-label` — 14px/500 with text-secondary color
- `.form-input` — dark input with border, focus ring animation (green glow)
- `.form-input:focus` — border transitions to `--border-focus`, adds `box-shadow: 0 0 0 3px rgba(52, 211, 153, 0.15)`
- `.form-section` — groups related fields with a subtle heading
- `.form-row` — side-by-side fields on desktop, stacked on mobile

Buttons:
- `.btn-primary` — white background, green text, hover scale, active press. Replaces `.btn-frosted`.
- `.btn-secondary` — outline style, transparent background with border
- `.btn-loading` — replaces text with a CSS spinner, maintains button width

Feedback:
- `.toast` — fixed bottom toast notification with slide-up animation
- `.toast-success`, `.toast-error` — color variants
- `.error-inline` — red-bordered card for inline form errors (replaces `alert()`)
- `.skeleton` — animated shimmer placeholder (pulsing gradient)
- `.skeleton-text`, `.skeleton-card` — preset skeleton shapes

Animations:
- `@keyframes fadeUp` — card entrance (opacity 0→1, translateY 12px→0)
- `@keyframes shimmer` — skeleton loading effect
- `@keyframes slideUp` — toast entrance
- `@keyframes spin` — button loading spinner
- `.animate-enter` — applies fadeUp with staggered `animation-delay` via CSS custom property

Typography:
- `.text-display` — 32px/700, letter-spacing: -0.02em
- `.text-heading` — 24px/600, letter-spacing: -0.01em
- `.text-body` — 16px/400
- `.text-label` — 14px/500, text-secondary
- `.text-caption` — 12px/400, text-tertiary

Spacing utilities:
- `.mt-{size}`, `.mb-{size}`, `.p-{size}`, `.gap-{size}` for xs through 3xl

### Task 2: Rewrite Home.js — Modern dashboard

**File:** `src/pages/Home.js` — Full rewrite.

**Requirements:**
- Use ONLY CSS classes from the design system. Zero inline styles.
- Page header with `.text-display` title "Inventory" and `.text-label` subtitle showing item count ("12 properties" or "No properties yet").
- Empty state: A centered card with an SVG illustration (simple building icon), heading, body copy per UI-SPEC contract, and a CTA button linking to #add.
- Loading state: 3 `.skeleton-card` placeholders with shimmer animation.
- Data state: Property cards using `.card.card-interactive` with:
  - Property name as card title
  - Location as subtitle
  - Price badge in top-right
  - Status dot indicator
- Error state: `.card` with destructive border, icon, and retry button.
- All cards use `.animate-enter` with staggered delays.

### Task 3: Rewrite main.js — Route transitions + toast system

**File:** `src/main.js` — Rewrite router and nav.

**Requirements:**
- Use CSS classes only. Zero inline styles.
- Add fade transition between routes (old content fades out, new fades in).
- Add a global `showToast(message, type)` function exported for use by form submission.
- Toast renders as a fixed-bottom notification that auto-dismisses after 3 seconds.
- Loading state uses skeleton placeholders instead of "Loading..." text.
- Error fallback uses design system classes.
- Bottom nav uses CSS classes for all styling.

## Verification

1. Dashboard renders with layered green surfaces (base → raised cards), not a flat single-color background.
2. Cards have visible borders, subtle shadows, and hover effects on desktop.
3. Loading shows shimmer skeleton placeholders, not plain text.
4. Empty state has an illustration and styled CTA button.
5. Error states use inline styled cards, not browser `alert()`.
6. All elements use CSS classes — `grep -rn "style=" src/pages/ src/main.js` returns zero matches.
7. Route transitions have a subtle fade effect.
8. Toast notification appears on successful form save.
