## Tasks

### Task 1: Deploy Firestore rules + verify database connectivity

The Firestore rules have been deployed (completed during planning). Verify the database `(default)` is active by adding a connection timeout to `inventoryService.js`.

**File:** `src/backend/inventoryService.js`

**Changes:**
- Wrap `getInventoryItems()` with a 8-second timeout using `Promise.race()`. If Firestore doesn't respond, resolve with an empty array and log a warning instead of hanging indefinitely.
- Wrap `createInventoryItem()` with a 10-second timeout. On timeout, throw a user-friendly error ("Save timed out. Please try again.") instead of hanging.

### Task 2: Fix PWA manifest generation

The `vite-plugin-pwa` only generates the manifest in the production build by default. For dev mode, the manifest needs explicit injection.

**File:** `vite.config.js`

**Changes:**
- Add `devOptions: { enabled: true }` to the VitePWA config so the manifest and service worker are available in dev mode.
- Add `icons` array with at least one 192x192 and one 512x512 icon entry (use placeholder paths — actual icons can be added later).

**File:** `index.html`

**Changes:**
- Add `<link rel="manifest" href="/manifest.webmanifest">` in the `<head>` section as a fallback for browsers that don't pick up the plugin's auto-injection.
- Add `<meta name="theme-color" content="#0a3622">` meta tag.

### Task 3: Fix bottom navigation visibility on mobile

The bottom nav uses `position: fixed; bottom: 0` which should work, but mobile browsers have viewport height issues with `100vh` and the URL bar.

**File:** `src/style.css`

**Changes:**
- Replace `min-height: 100vh` on `body` with `min-height: 100dvh` (dynamic viewport height) with a fallback: `min-height: 100vh; min-height: 100dvh;`
- Ensure `body` has `padding-bottom: calc(70px + env(safe-area-inset-bottom))` to account for iOS safe area.
- Update `.bottom-nav` to add `padding-bottom: env(safe-area-inset-bottom)` for notched devices.
- Add `height: auto; min-height: 70px;` to `.bottom-nav` so it doesn't get clipped.

**File:** `index.html`

**Changes:**
- Add `viewport-fit=cover` to the existing viewport meta tag to enable safe area insets.
