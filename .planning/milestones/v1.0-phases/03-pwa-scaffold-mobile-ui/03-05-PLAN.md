## Tasks

### Task 1: Rewrite IntakeForm.js with all property fields

**File:** `src/pages/IntakeForm.js` — Full rewrite.

**Requirements:**
- Use ONLY CSS classes from the design system (from Plan 03-04). Zero inline styles.
- Organize the 23 fields into logical sections using `.form-section`:

**Section 1: Property Information**
1. Property Name — text input, required
2. Building Type — select dropdown: Mall, Standalone, High Street
3. Which Floor — text input
4. Exact Location / Address — text input, required
5. Trade Area — text input  
6. Suitable For — text input (e.g., "F&B, Retail, Services")

**Section 2: Contact Details**
7. Contact Name — text input, required
8. Contact Designation — text input
9. Contact Info (Phone/Email) — text input, required

**Section 3: Unit Specifications**
10. Size of Unit (sq ft) — number input, required
11. Price per Sq Ft (Carpet) — number input, required
12. CAM (per sq ft) — number input
13. Clear Height (ft) — number input
14. Connected Load (KW) — number input
15. Age of Building (years) — number input

**Section 4: Facilities**
16. Parking Space — yes/no toggle + number input for count
17. Outside Visibility — yes/no toggle
18. Service Entry — yes/no toggle
19. Lift Access — yes/no toggle
20. BOH Space — yes/no toggle
21. Fire Exit — yes/no toggle
22. OC File Available — yes/no toggle

**Section 5: Photos & Documents**
23. Building Facade — file input (multiple images), camera capture on mobile
24. Unit Facade — file input (multiple images)
25. Interior — file input (multiple images)
26. Signage — file input (multiple images)
27. Floor Plan — file input (image or PDF)
28. Parking Photo — file input (conditional, shown only if Parking = yes)
29. Service Entry Photo — file input (conditional, shown only if Service Entry = yes)
30. Lift Access Photo — file input (conditional, shown only if Lift Access = yes)
31. BOH Space Photo — file input (conditional, shown only if BOH Space = yes)

**UX details:**
- Each section has a `.form-section` heading (e.g., "Property Information") and is collapsible on mobile (starts expanded for Section 1, collapsed for others).
- Yes/No toggles render as pill buttons (two-option segmented control), not checkboxes.
- File inputs show a dashed-border upload zone with camera icon. On mobile, offer camera capture via `accept="image/*" capture="environment"`.
- File previews show as thumbnail grid below the upload zone after selection.
- Submit button shows loading spinner via `.btn-loading` class.
- On success: call `showToast('Property registered successfully!', 'success')` from main.js, then navigate to `#`.
- On error: show `.error-inline` card below the submit button with the error message. No `alert()`.

### Task 2: Update inventoryService to handle photo uploads

**File:** `src/backend/inventoryService.js`

**Changes:**

**Changes:**
