## Objective

Create a Cloud Function that serializes the inventory collection to a JSON file in Firebase Storage, accessible via a public URL. The main website fetches this static file. No dynamic API needed.

## Tasks

### Task 1: Initialize Firebase Functions in the project

**Changes:**
- Run `npx -y firebase-tools@latest init functions` (select JavaScript, ESLint optional).
- This creates a `functions/` directory in the project root.
- Configure it to use the `footfall-inventory` project.

### Task 2: Write the export function

**File:** `functions/index.js`

**Requirements:**
- Create a Cloud Function `exportInventoryToJSON`:
  - Triggered by an HTTP request (callable via URL or schedulable via Cloud Scheduler).
  - Reads all documents from the `inventory` collection.
  - For each document, strips internal fields (`created_at`, `updated_at` timestamps are converted to ISO strings).
  - Builds a JSON object: `{ lastExported: ISO timestamp, count: N, properties: [...] }`.
  - Uploads the JSON to Firebase Storage at `exports/inventory.json` with `public-read` ACL and `application/json` content type.
  - Returns `{ success: true, count: N, url: publicUrl }`.

- Create a scheduled version `scheduledExportInventory`:
  - Runs daily at 2:00 AM IST (`schedule('0 2 * * *', { timeZone: 'Asia/Kolkata' })`).
  - Calls the same export logic.
