# Plan 04-01: Photo Upload Pipeline

> **Wave 1** — The intake form already has file input zones but photos are not uploaded. Wire the Storage service into the submission flow.

## Objective

Connect the IntakeForm's file inputs to Firebase Storage via `storageService.js`, so photos are uploaded during property registration and their download URLs are persisted in the Firestore document.

## Tasks

### Task 1: Add uploadMultipleFiles helper to storageService.js

**File:** `src/backend/storageService.js`

**Changes:**
- Add `uploadMultipleFiles(files, basePath)` — accepts a `FileList` or `File[]`, uploads each to `{basePath}/{filename}`, and returns an array of download URLs.
- Use `Promise.allSettled()` for resilience — if one photo fails, others still succeed.
- Add a progress callback that reports total aggregate progress across all files.

### Task 2: Wire photo uploads into IntakeForm submission

**File:** `src/pages/IntakeForm.js`

**Changes:**
- After `createInventoryItem(data)` returns the document ID, collect all file inputs that have files selected.
- For each photo category (buildingFacade, unitFacade, interior, signage, floorPlan, conditional photos like parkingPhoto, serviceEntryPhoto, liftAccessPhoto, bohSpacePhoto):
  - Call `uploadMultipleFiles(files, 'properties/{docId}/{category}')`.
  - Collect the returned download URLs.
- Call `updateInventoryItem(docId, { images: { buildingFacade: [...urls], unitFacade: [...urls], ... } })` to attach URLs to the document.
- Show upload progress in the UI: replace "Saving..." with a progress percentage.
- If photo upload partially fails, show a toast warning ("Property saved but some photos failed to upload") instead of blocking.

### Task 3: Display photos in property card (Home.js)

**File:** `src/pages/Home.js`

**Changes:**
- If a property has `images.buildingFacade[0]`, show it as a thumbnail in the card header (small 48x48 rounded image).
- Add a subtle camera icon badge if item has photos, count badge showing total photo count.

## Verification

1. Register a property with photos → property card shows thumbnail + photo count badge.
2. Register a property WITHOUT photos → card shows without thumbnail (graceful fallback).
3. Partially fail upload (e.g., disconnect mid-upload) → toast warning, property still saved with text data.
