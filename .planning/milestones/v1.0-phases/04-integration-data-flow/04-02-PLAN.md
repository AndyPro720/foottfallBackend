# Plan 04-02: Property Detail View

> **Wave 1** — Users can list properties but can't view full details. Add a detail route with all 23 fields + photo gallery.

## Objective

Create a Property Detail page accessible via `#property/{id}` that shows all saved data for a registered property, including a photo gallery.

## Tasks

### Task 1: Create PropertyDetail.js page component

**File:** `src/pages/PropertyDetail.js` — **NEW**

**Requirements:**
- Fetch property by ID using `getInventoryItemById(id)`.
- Display all fields organized into the same 5 sections as the intake form:
  - **Property Info**: name, building type, floor, location, trade area, suitable for
  - **Contact**: contact name, designation, phone/email
  - **Specs**: size, price/sqft, CAM, clear height, connected load, building age
  - **Facilities**: all toggle fields rendered as pills (Yes/No visual indicators)
  - **Photos**: full photo gallery with lightbox-style view
- Use design system classes exclusively.
- Include a "Back" button in the page header that navigates to `#`.
- Show a skeleton loader while fetching.
- Error state if property not found (404 card).

### Task 2: Photo gallery component

**Within:** `src/pages/PropertyDetail.js`

**Requirements:**
- Display photos grouped by category (Building Facade, Unit Facade, Interior, etc.).
- Each category shows a horizontal scrollable row of thumbnail images.
- Tapping a thumbnail opens a full-screen overlay with the image (simple lightbox).
- Lightbox has close button and swipe between images in the same category.
- If no photos exist for a category, hide that category.

### Task 3: Wire detail route into router

**File:** `src/main.js`

**Changes:**
- Add route handling for `#property/{id}` hash pattern.
- Extract the document ID from the hash and pass it to `renderPropertyDetail(container, id)`.

### Task 4: Make dashboard cards clickable

**File:** `src/pages/Home.js`

**Changes:**
- Wrap each property card in an `<a href="#property/{item.id}">` link.
- Cursor pointer is already handled by `.card-interactive`.

## Verification

1. Click a property card on the dashboard → navigates to `#property/{id}` and shows all details.
2. Photo gallery shows categories with scrollable thumbnails.
3. Tap a thumbnail → full-screen lightbox opens.
4. Back button → returns to dashboard.
5. Navigate to `#property/nonexistent-id` → shows "Property not found" card.
