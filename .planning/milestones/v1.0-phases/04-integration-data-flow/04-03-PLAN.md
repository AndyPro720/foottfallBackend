# Plan 04-03: Edit, Delete & Status Management

> **Wave 2** — After detail view exists (Wave 1), add the ability to edit properties, delete them, and manage their status.

## Objective

Complete the CRUD loop: users can edit existing properties, delete them with confirmation, and change status (active/pending/inactive).

## Tasks

### Task 1: Edit Property page

**File:** `src/pages/EditProperty.js` — **NEW**

**Requirements:**
- Route: `#edit/{id}`.
- Reuse the same `SECTIONS` field definitions from IntakeForm.js (extract to a shared config file `src/config/propertyFields.js`).
- Pre-populate all form fields with existing property data on load.
- For file/photo fields, show existing uploaded photos as thumbnails with "×" remove buttons and the option to add more.
- Submit button reads "Save Changes" instead of "Register New Retail Property".
- On save: call `updateInventoryItem(id, data)` and show success toast.
- Navigation: accessible from the PropertyDetail page via an "Edit" button.

### Task 2: Extract shared field definitions

**File:** `src/config/propertyFields.js` — **NEW**

**Changes:**
- Move the `SECTIONS` array from IntakeForm.js into this shared module.
- Export it so both IntakeForm.js and EditProperty.js import from the same source.
- Update IntakeForm.js to import from this shared config.

### Task 3: Delete with confirmation

**File:** `src/pages/PropertyDetail.js`

**Changes:**
- Add a "Delete Property" button at the bottom of the detail page, styled as `.btn-secondary` with destructive color.
- On click, show a confirmation modal:
  - Heading: "Delete Property"
  - Body per UI-SPEC: "Are you sure you want to remove this item? This action is permanent."
  - Two buttons: "Cancel" (secondary) and "Delete" (destructive primary).
- On confirm: call `deleteInventoryItem(id)`, show toast "Property deleted", navigate to `#`.

### Task 4: Status management

**File:** `src/pages/PropertyDetail.js`

**Changes:**
- Show current status as a colored badge at the top (green=active, yellow=pending, gray=inactive).
- Add a status dropdown/toggle that allows changing between Active, Pending, Inactive.
- On status change, call `updateInventoryItem(id, { status: newStatus })`.

### Task 5: Wire edit route

**File:** `src/main.js`

**Changes:**
- Add `#edit/{id}` route.

## Verification

1. Edit an existing property → fields pre-populated → save → toast "Property updated" → detail view shows new data.
2. Delete a property → confirmation modal → confirm → toast "Property deleted" → redirected to dashboard → property gone.
3. Change status from Active to Inactive → badge color changes → reload → status persists.
