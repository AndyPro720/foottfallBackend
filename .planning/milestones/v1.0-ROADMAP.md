# ROADMAP.md

> **Current Milestone**: Inventory Application Backend
> **Goal**: Build a mini app/pwa to get inventory intake(retail units) and all pertaining data on the go along with a backend to store it and showcase it. The website will also pick up from this backend. Backend to pave way the entire system backend for future which may include crm, etc.

## Must-Haves
- [x] Fluid, mobile compitable and agile
- [x] Backend must take larger picture into account
- [x] Integrate with exising backend as well

## Nice-To-Haves
- [x] User log ins
- [x] Works without internet

## Phases

### Phase 1: Architecture & Backend Foundation
**Status**: ✅ Complete
**Objective**: Design the database schema and API structure, ensuring it accommodates the "larger picture" (CRM, etc.) and integrates smoothly with the existing backend.

### Phase 2: Core API Development
**Status**: ✅ Complete
**Objective**: Build out the necessary endpoints for inventory intake (retail units) and data retrieval, providing the infrastructure for both the PWA and the website.

### Phase 3: PWA Scaffold & Mobile UI
**Status**: ✅ Complete
**Objective**: Create the fluid, mobile-first frontend application focused on agility and on-the-go data entry.

### Phase 4: Integration & Data Flow
**Status**: ✅ Complete
**Objective**: Connect the PWA to the new backend APIs. Simultaneously, wire up the main website to read from this new central inventory storage.

### Phase 5: Offline Capabilities & Authentication
**Status**: ✅ Complete
**Objective**: Implement user logins and service workers for offline functioning (syncing data when internet is restored).

### Phase 6: Verification & Polish
**Status**: ✅ Complete
**Objective**: End-to-end testing, responsive design polish, and final cross-platform checks before deployment.
